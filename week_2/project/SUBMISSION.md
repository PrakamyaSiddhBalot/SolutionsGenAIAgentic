## ResearchBot – Week 2 Project 

For this project, I built a research assistant that can search the web, read webpages, and look through research papers using AlphaXiv. The final version includes a terminal-based interface where users can chat with the assistant, see which tools it is using, and ask research questions in a more interactive way. It was interesting to see how a simple chatbot could gradually evolve into something that actually gathers information before answering.

The way my agent works is fairly straightforward. When a user asks a question, the model first decides whether it already knows enough to answer. If it needs more information, it can use one of the available tools. After the tool returns information, the model looks at the new information and decides whether it needs to keep researching or whether it can finally answer the user. This process continues until it feels it has enough evidence to respond.

One design decision I made was to separate the backend and the user interface into different files. This made the project much easier to manage as it grew. I also spent some time making sure the interface stayed responsive while the agent was working. My first version would completely freeze whenever the model was searching for information, which made the application feel broken. Fixing that made a huge difference to the user experience.

AlphaXiv was the biggest surprise of the project. After reading the documentation I thought that connecting an MCP server would be a simple process, but it turned out to be the most time consuming part of the whole project. The AlphaXiv chatbot kept providing information that was incomplete or just plain wrong. At one point it kept telling me to generate an API key from a settings page that didn't even exist on my account. I went down a few dead ends before I finally realized that browser authentication was how I was supposed to be doing this.

Another challenge was figuring out how my MCP library was supposed to communicate with AlphaXiv. The examples I was reading did not quite match what I had installed, so I spent a long time experimenting before I finally got a successful connection. It was one of those moments where nothing seemed to work until suddenly everything clicked into place.

The most frustrating issue was the authentication token. The token generated by the browser login only lasted about 57 seconds. That meant if I spent too long debugging something else, the token would expire and I would start getting errors again. More than once I thought I had broken my code when the real problem was simply that the login had expired.

Given more time, I would improve the authentication process so the agent could automatically detect expired logins and reconnect when necessary. I would also like to make the tool activity log more informative and improve the overall polish of the interface. Overall, this project taught me that building the features is often the easy part. The real challenge is getting different systems to work together reliably.
